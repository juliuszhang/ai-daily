## 社区热议

### 1. Qwen3.8 开源预告引发本地推理社区高潮
@Alibaba_Qwen 官方发布 Qwen3.8（2.4T 参数、即将开源权重）后，社区迅速讨论其在本地推理、编码、工具调用场景的潜力。多位开发者表示 "GPT OSS 已成过去式，现在只剩 Qwen 和 Gemma4"，并期待更小参数变体发布。Qwen3.8-Max-Preview 已可通过阿里 Token Plan、Qoder、QoderWork 访问。来源：、

### 2. Claude Code 18,000-agent 编排系统视频广传
一段被广泛分享的演讲视频拆解了 Claude Code 团队构建的 **18,000 个 agent、140,000 步** 的 agent loop 系统，约 90% 工作无需人工输入。关键点：从原始 API 调用转向托管 agent（交付速度提升 10-15×）、P95 延迟下降 >90%、Claude 自主定位根因、子 agent + 持久记忆 + "dreaming" + vault 机制用于长周期自主任务。社区称其 "比多数付费 agent 课程更有价值"。
### 3. "Claude Agent OS" 编排概念兴起
社区讨论一种统一编排层：廉价任务路由到低成本模型（如 GLM，编码价格约 Claude 的 1/6），困难任务路由到 Claude，重大决策交由 "辩论面板 + 裁判" agent，所有 agent 共享 "大脑"/笔记本以保持上下文一致。目标：睡前交接任务，醒来拿到成品。
### 4. Elon Musk 为 Grok 4.5 与 Grok Build 背书
Elon 在 X 上引用 Grok 4.5 在 VulcanBench 取得 91.3%（超过 Claude、GPT 模型，真实编码任务）的基准结果，回复 "Try Grok 4.5!"；另引用 Grok 成功预测西班牙 2026 FIFA 世界杯夺冠的旧帖，称 "Grok is good at predictions"；并针对 **Grok Build 0.2.105**（将 Grok 4.5 设为默认模型、新增 `/summarize` 命令、改进 shell 工具与会话处理）回复 "Try Grok Build."。来源：、

### 5. 中国实验室开源前沿模型浪潮（社区传闻/未确认）
社区多条帖子提及中国实验室即将发布的大模型，**均未由对应公司官号在 24 小时内确认**，暂列为社区传闻：
- **DeepSeek V4**：传闻即将 GA，1T MoE、1M 上下文、~$0.0028/M "Peak-Valley" 定价、SWE-Bench 80%+，预期超越 Opus。
- **智谱 GLM-5.2 / GLM-5.5**：传 GLM-5.2 接近 Claude Opus 水平，GLM-5.5 预期进一步超越。

- **Moonshot Kimi K3 / KimiWork**：传接近 Fable 5 水平，2.8T 规模。

## 总体趋势观察

- **开源前沿模型再加速**：阿里 Qwen3.8（2.4T、即将开源）是本日唯一由官号正式发布的产品级动态，也是社区讨论的最大焦点。结合社区传闻中的 DeepSeek V4、GLM-5.5、MiniMax M3 Pro、Kimi K3，中国实验室正以 "大参数 + 开源/低价" 持续挤压闭源前沿模型的经济空间。
- **Agent 编排成为主流叙事**：Claude Code 的 18k-agent 系统视频与 "Claude Agent OS" 编排概念表明，社区关注点已从单模型能力转向多 agent 协调、任务路由、持久记忆与安全审计层。Hermes Agent 被 OpenClaw 采用也印证了这一方向。
- **官号沉默日**：OpenAI、Anthropic、Google DeepMind、SpaceXAI、DeepSeek、MiniMax、智谱、OpenClaw 八家官号在过去 24 小时内均无实质产品/研究动态；Elon Musk 以个人账号为 Grok 4.5 与 Grok Build 背书，但 @SpaceXAI 官号本身未发文。
- **AI 审计 AI 成为新竞争点**：随着 AI 生成代码进入生产环境，社区开始系统性讨论 "AI 检查 AI" 的安全与审查层，Codex 与 Claude Code 配对交叉验证成为常见工作流。
