## 社区热议

- **Claude Code Desktop 加入应用内浏览器——官方已确认。** Claude 开发者官方账号 @ClaudeDevs 表示，Claude 可在桌面应用中打开外部文档、设计稿和网站，读取页面、点击并执行交互；浏览环境采用沙箱隔离，用户可配置会话是否保留。

- **Perplexity 接入 Grok 4.5 作为代理编排模型——官方已确认。** Consumer Pro 与 Max 用户可在 Perplexity Computer 中选用 Grok 4.5。Perplexity 称其在六种编排配置的 WANDR 测试中得分最高，成本约为 Claude Opus 4.8 的一半。

- **Elon Musk 集中转发 Grok 4.5 与 Grok Build 的应用数据。** 其帖子提及 Grok 4.5 在 Perplexity 的编排模型接入，并引用 Grok Build 在 SWE‑Atlas‑QnA 上取得 **84 分**的结果；相关产品发布并非来自本期受监控的 @SpaceXAI 官号，因此未计入公司动态表。

- **独立开发者发布 Qwen3.6‑35B‑A3B AntiLoop 轻量 LoRA。** 发布者称其针对困难提示下的循环输出进行优化，自测循环率降低约 **7 倍**，并报告 GPQA、GSM8K 表现基本保持；这些指标为发布者自报，并非阿里/Qwen 官号结果。

## 总体趋势观察

1. **模型竞争继续转向“效果—成本”联合指标。** OpenAI 直接强调 GPT‑5.6 Luna 相对 GPT‑5.5 的性能与成本差异，Perplexity 对 Grok 4.5 的评价也同时披露编排效果和单次任务成本。

2. **编码代理正在形成完整的操作闭环。** Claude Code 的应用内浏览器覆盖外部资料读取与网页交互；Hermes Desktop 则将本地桌面入口与云端常驻代理连接起来，产品边界从“生成代码”扩展至“查看、操作、验证和持续运行”。

3. **代理编排由单模型执行转向按任务选择模型。** Grok 4.5 被作为 Perplexity Computer 的编排器接入，社区中同时出现面向 Codex、Claude Code、Grok 等工具的路由与组合工作流。

4. **能力扩展与安全治理同步推进。** OpenAI 将生物安全漏洞赏金计划长期化，Google DeepMind集中讨论思维链监控、机械可解释性和模型审计，表明前沿能力发布正伴随更制度化的安全测试与内部机制研究。
