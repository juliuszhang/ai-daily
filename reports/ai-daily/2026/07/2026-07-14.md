## 社区热议

### 1. GPT-5.6 三型号社区评测持续深入

GPT-5.6 于约 7 月 9 日发布（超出 24h 窗口），但社区评测和基准分析在过去 24 小时内持续活跃：

- **Sol（旗舰）**：在 Artificial Analysis Coding Agent Index 达 SOTA ~80，Terminal-Bench 2.1 得分 91.9%（Ultra 模式）；部分编码任务 token 效率提升 54%。定价 $5/$30（输入/输出，每百万 token）。
- **Luna（轻量）**：某些场景下以约 40% 更低成本匹配 Sol 输出质量，适合高量轻任务。定价 $1/$6。Token 用量更高、推理速度约为 Sol 的 2 倍慢。
- **Terra（中端）**：社区评价偏负面，认为在 Pareto 前沿位置尴尬，多数场景下 Luna 或 Sol 更优。定价 $2.5/$15。
- 三款均获「High」安全评级，未达「Critical」风险阈值。

### 2. OpenAI 开源 Codex Claude Code 插件（官方已确认）

OpenAI 在 GitHub 开源了 Codex Claude Code 插件（仓库 `openai/codex-plugin-cc`），支持在 Claude Code 内直接调用 Codex 能力：

- `/codex:review` — 获取代码审查第二意见
- `/codex:adversarial-review` — Codex 对抗性审查实现和假设
- `/codex:rescue` — Claude 卡住时委托 Codex 接手
- `/codex:transfer` — 将当前会话移交 Codex 继续工作

安装方式：`/plugin marketplace add openai/codex-plugin-cc` → `/plugin install codex@openai-codex` → `/reload-plugins` → `/codex:setup`（需 Node 18.18+ 及 ChatGPT 订阅或 OpenAI API key）。

来源：[GitHub openai/codex-plugin-cc](https://github.com/openai/codex-plugin-cc)，社区讨论

### 4. Anthropic J-space / J-lens 隐式推理研究讨论

社区讨论 Anthropic 内部「J-space」（心理工作空间）研究：使用 J-lens 工具可观测到 LLM 内部约 25 个关键概念在输出中不显现但影响输出的隐式多步推理。社区认为这对检测偏见、欺骗和提升代理可靠性有重要意义。（此研究在社区中讨论活跃，@AnthropicAI 官号在过去 24h 未单独发布 J-space 相关帖。）

### 6. Grok 编码代理遥测争议（社区传闻/未确认）

有社区审计声称 Grok 编码代理每轮将整个代码库上传至 Google Cloud Storage，而 Claude 和 Codex 据报不会。该说法未经 SpaceXAI 官方确认。Elon Musk 同期宣布的彻底删除历史用户数据政策或与此相关，但官方未直接回应遥测指控。


### 7. Agent 上下文管理与过夜工作流

- **上下文膨胀**：开发者讨论长时间运行代理的上下文污染问题，建议限制会话历史至最近 15 次、裁剪已完成块、状态文件控制在 ~200 行 / 500KB 以内。
- **过夜代理**：开发者分享使用开源工具栈（RuFlow、gstack、ai-memory-vault）在 Qwen、DeepSeek、Kimi、GLM 等便宜模型上运行代理，实现近零成本过夜代码生成。

## 总体趋势观察

1. **隐私与数据主权成为竞争焦点**：SpaceXAI 主动宣布 ZDR 和追溯删除，Elon Musk 亲自背书，将隐私保护提升为核心卖点。这与社区对 Grok 编码代理遥测行为的审计形成对照，表明数据隐私正从合规要求转向产品差异化竞争维度。

2. **AI 编码工具走向可组合化**：OpenAI 开源 Codex Claude Code 插件、社区讨论多代理协作工作流、过夜代理栈——行业正从「选一个模型」转向「多模型协同」，不同代理各司其职。Claude Code 的插件市场和 Skills 体系正在成为这种可组合性的基础设施。

3. **模型分层定价常态化**：GPT-5.6 的 Sol/Terra/Luna 三级定价、DeepSeek V4-Pro/Flash 分层、Claude Fable 5 限额调整——各家都在用能力分层匹配任务复杂度，降低每任务成本成为核心竞争指标。

4. **Agent 工程实践成熟化**：上下文卫生管理、会话历史裁剪、状态文件控制等经验在社区快速沉淀为最佳实践，表明 Agent 从实验阶段进入生产阶段的工程门槛正在被系统性地解决。

5. **中国 AI 官号静默期**：智谱、DeepSeek、MiniMax、阿里 Qwen 在 X 上的官号在过去 24h 均无实质产品/研究动态发布，与 OpenAI/Anthropic/SpaceXAI 的活跃形成对比。
